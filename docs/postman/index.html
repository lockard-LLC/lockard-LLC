Postman quick start

Connect Postman to the Lockard LLC API discovery assets, validate your setup, and keep collections in sync with our OpenAPI schema.

Get your API key

  1. Visit Postman API keys.
  2. Generate a key and copy the value (format: PMAK-…).
  3. Create .env from .env.example and add: 
    POSTMAN_API_KEY=PMAK-your-key-here

Run the setup helper

Install dependencies and execute the helper script from the repository root:

yarn install
yarn postman:setup
# or
npm install
npm run postman:setup

The helper verifies your API key, lists available workspaces, and summarizes the configuration files that ship with this repo.

Import into Postman

Option A — Postman Web UI

  1. Open Postman and go to APIs → Create API → Import.
  2. Select File and choose docs/postman/schemas/openapi.yaml.
  3. Set the collection directory to docs/postman/collections/ and click Import.

Option B — Git integration

  1. In Postman, choose APIs → Create API → Connect Repository.
  2. Connect to GitHub and select the main branch.
  3. Set the root file to docs/postman/schemas/openapi.yaml and the collection directory to docs/postman/collections/.

Discovery endpoints

  • APIs.json: /.well-known/apis.json
  • OpenAPI schema: docs/postman/schemas/openapi.yaml
  • Postman collection: /.well-known/postman.json

Reference commands

# Validate .well-known catalog
yarn wellknown:validate

# Postman helper
yarn postman:setup

# Local preview
yarn dev

# Deploy hosting artifacts
yarn deploy

Customize the API

Add an endpoint in docs/postman/schemas/openapi.yaml:

paths:
  /your-endpoint:
    get:
      summary: Your endpoint description
      responses:
        '200':
          description: Success response

Model new data structures under components.schemas and extend components.securitySchemes to register additional auth strategies.

Troubleshooting

  • API key errors? Confirm POSTMAN_API_KEY is present in .env.
  • Collections missing? Verify the directory is set to docs/postman/collections/.
  • Schema updates? Re-run yarn postman:setup after editing openapi.yaml.
